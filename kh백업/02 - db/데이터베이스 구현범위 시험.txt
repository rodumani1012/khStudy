선다 5개 서술 2개

1. 우리가 설치하고 있는 dbms - oracle
express edition : 단일 인스턴스만 사용 가능, 메모리 최대 1GB, 무료(데모, 교육용), xe(express edition의 이름)
standard edition : 기업용 일반용, orcl(standard edition의 이름)

2. 다음 4개의 정의 중에 하나 묻는 문제.
Data Dictionary - db에 저장되는 데이터에 관한 정보를 저장하는 곳  Database - 특정 조직이 업무 수행을 위해 모은 자료집합체  Tablespace - 실제 데이터 저장 공간 Procedure - 특정 작업을 수행하기 위한 블록

3. 스키마 : 데이터베이스의 구조, 제약조건에 관한 전반적인 명세를 기술 - 외부. 사용자 관점 개념. db의 전체적인 논리적 구조 내부. 물리적 구조 - 메타데이터라고도 부름.

4. 제약조건 : not null은 값이 항상 입력되어야 할 때 사용. 제약조건은 한번에 여러개 적용 가능 	인덱스 별도 설정 필요 없음 check는 컬럼에 들어오는 값을 제한하고 싶을 때.

5. 스크립트 4개(select from ~~~) 나오는데 결과가 다른 한 개 찾는 문제.

6. 테이블과 인덱스에 대해 설명하시오
- 테이블 : rdbms에서 정렬된 데이터 집합의 모임
- 인덱스 : 테이블의 동작 속도를 높여주는 자료구조

7. 관리자 계정과 사용자 계정의 역할
- 관리자 계정 : 데이터베이스의 생성과 관리를 담당하는 슈퍼 유저 계정이며, 정의, 생성, 삭제 등의 
작업이 가능하다. 데이터베이스에 대한 모든 권한과 책임을 가지는 계정이다. 
오라클 데이터베이스 생성 시 관리자 계정인 SYS와 SYSTEM 계정이 자동으로 생성되 
어 있다. MySQL 데이터베이스인 경우에는 관리자 계정 root가 기본적으로 생성된다.

- 일반 계정 : 데이터베이스에 대하여 질의, 갱신, 보고서 작성 등의 작업을 수행할 수 있는 계정이다. 일반 계정은 업무에 필요한 최소한의 권한만 가지는 것을 원칙으로 한다.
